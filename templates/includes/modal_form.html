{% load i18n %} {% comment %} Shared shell for every create / edit / confirm form. Rendered into #drawer-form-content (the .modal-content of the global #itemDrawer modal). Two modes: - Declarative (form mixes in core.modal_forms.SteppedFormMixin): the shell auto-renders the stepper / completion meter and the fields from the form's `steps`. Entity templates only set the header icon. - Legacy: the entity template overrides `modal_form_fields` with hand-written markup; the progress bar and step footer stay off because `form.is_stepped` is falsey. {% endcomment %}
{% comment %} novalidate on multi-step forms: native browser validation can't focus a field on a hidden step ("invalid form control is not focusable"), which would block submit. Validation is gated per step in JS and enforced server-side (the source of truth), which re-renders the modal on error. {% endcomment %} {% csrf_token %}
